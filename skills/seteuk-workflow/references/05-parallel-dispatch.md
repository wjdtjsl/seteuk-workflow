# 5. 초안 작성과 최종 점검의 서브에이전트 병렬 처리 기준

2단계(초안 작성)와 3단계(최종 점검) 모두 이 절차를 따른다. 학생 수가 많을 때 한 명씩 순서대로 처리하면 시간이 오래 걸리므로, 학생을 여러 그룹으로 나눠 서브에이전트(Agent 도구)에게 동시에 맡긴다.

## 서브에이전트 개수 결정

작업 대상 학생 수를 센 뒤 아래 표로 서브에이전트 개수를 정한다.

| 학생 수 | 서브에이전트 수 | 서브에이전트당 학생 수(대략) |
|---|---|---|
| 1~3명 | 사용 안 함 (직접 순서대로 처리) | - |
| 4~6명 | 2개 | 2~3명 |
| 7~10명 | 3개 | 3~4명 |
| 11~15명 | 4개 | 3~4명 |
| 16~20명 | 5개 | 4명 |
| 21명 이상 | 6개 (상한) | 4명 이상, 균등 분배 |

1~3명은 분배, 프롬프트 구성, 결과 취합에 드는 오버헤드가 병렬 처리로 아끼는 시간보다 크므로 서브에이전트 없이 직접 처리한다. 6개를 상한으로 두는 이유는, 동시 실행 개수가 늘어날수록 결과 취합과 오류 처리가 더 복잡해지고 서브에이전트당 4명 이상을 맡겨도 개별 학생 처리 품질에 큰 지장이 없기 때문이다. 이 표는 기준일 뿐이며, 학생별 원자료 분량이 유난히 많으면 서브에이전트당 인원을 더 줄여도 된다.

## 학생 분배

학생을 서브에이전트 수만큼 최대한 균등하게 나눈다. 나머지 인원은 앞쪽 그룹부터 1명씩 추가한다(예: 학생 11명, 서브에이전트 4개면 3, 3, 3, 2명으로 나눈다).

## 서브에이전트 프롬프트 구성

Agent 도구로 서브에이전트를 호출할 때 subagent_type은 general-purpose를 사용한다. 각 프롬프트는 그 자체로 완결되어야 한다 — 서브에이전트는 지금까지의 대화 맥락을 전혀 알지 못하므로, 아래 내용을 프롬프트 안에 모두 포함한다.

1. **작업 정의**: 지금 하는 일이 세특 작성인지 점검인지, 교과목/학년/학기, 목표 글자수 또는 바이트수, 세특 강조점(있다면).
2. **규칙 전문**: 작성 단계는 `references/02-writing.md`, 점검 단계는 `references/03-check.md`와 `references/04-prohibited-and-format.md`의 핵심 규칙을 프롬프트에 요약해 포함하거나, 서브에이전트가 해당 파일을 직접 Read하도록 절대경로(`${CLAUDE_PLUGIN_ROOT}/skills/seteuk-workflow/references/...`)를 안내한다. 규칙을 누락하면 서브에이전트가 규정을 어긴 결과를 낼 수 있으므로 반드시 포함한다.
3. **담당 학생 자료만 포함**: 이 서브에이전트가 담당하는 학생들의 자료(작성 단계는 1단계 정리 결과, 점검 단계는 2단계 초안)만 프롬프트에 담는다. 다른 그룹 학생의 자료는 섞지 않는다 — 그룹 간 정보가 섞이면 세특 개별화 원칙이 깨진다.
4. **워크플로우 전체 원칙 재전달**: 근거 없는 내용을 지어내지 않을 것, 판단이 애매하면 만들어내지 말고 "확인필요"로 표시해 돌려줄 것, 가운데점을 비롯한 특수문자를 쓰지 않을 것을 프롬프트에 명시한다. 서브에이전트는 이 워크플로우의 지침을 자동으로 물려받지 않는다.
5. **출력 형식 지정**: 아래처럼 학생별로 명확히 구분된 형식으로만 응답하도록 지시한다. 형식이 흐트러지면 결과 취합 단계에서 파싱이 어려워진다.

```
### [학번 또는 이름]
[완성된 세특 문단 또는 최종본]
글자수: N자 / 바이트수: M바이트
[점검 단계에서는 이어서] 수정 사항: [1~2줄 요약, 또는 "이상 없음"]
[확인이 필요하면] 확인필요: [무엇을 되물어야 하는지]
```

6. **검증 스크립트 안내**: Bash 도구로 `${CLAUDE_PLUGIN_ROOT}/skills/seteuk-workflow/scripts/check_bytes.py`, `check_seteuk.py`, `byte_counter.py` 같은 스크립트를 직접 실행해 글자수와 금지 표현을 스스로 검증한 뒤 응답하도록 안내한다.

## 동시 실행

구성한 프롬프트로 Agent 도구를 서브에이전트 개수만큼 호출하되, 서로 의존관계가 없으므로 반드시 한 메시지 안에서 여러 번 호출해 동시에 실행한다. 하나씩 순서대로 호출하면 병렬 처리의 이득이 사라진다.

## 결과 취합

모든 서브에이전트의 응답이 돌아오면, 학생별 항목을 파싱해 원래 학생 순서대로 정리한다.

- 응답에서 학생이 누락되었거나 형식이 어긋나 파싱할 수 없는 경우, 해당 학생만 별도로 하나의 서브에이전트를 다시 호출하거나 직접 처리한다. 전체를 다시 돌리지 않는다.
- "확인필요"로 표시된 학생이 있으면 취합 후 한꺼번에 모아 사용자에게 되묻는다. 서브에이전트끼리는 서로 통신할 수 없으므로 이런 조율은 항상 오케스트레이터(메인 에이전트)가 담당한다.
- 취합 후 전체 학생 수가 원래 대상 학생 수와 일치하는지 반드시 확인한다.
- 작성 단계 결과는 `세특초안_정리.xlsx`로, 점검 단계 결과는 `세특_최종.xlsx`로 저장한다. 파일 형식은 `references/02-writing.md`, `references/03-check.md`에 정의된 엑셀 형식을 그대로 따른다.

## 사용자에게 보고

각 단계가 끝나면 서브에이전트를 몇 개 사용했는지, 각 서브에이전트가 몇 명을 담당했는지 한 줄로 짧게 언급한다. 과정을 장황하게 설명하지 않는다.
