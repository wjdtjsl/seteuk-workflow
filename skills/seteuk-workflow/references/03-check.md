# 3단계: 최종 점검과 자연스러움 마무리 (모든 교과, 모든 학년 공통)

세특은 교사가 수업 중 직접 관찰한 사실을 근거로 나이스(NEIS) 표준 서식에 맞춰 작성하는 공식 기록이다. 2단계에서 나온 초안이 아무리 잘 쓰였어도 다음 두 가지 문제 중 하나라도 있으면 문제가 된다.

첫째, 규정 문제다. 관찰 근거 없는 단정, 기재금지 항목, 서식 위반, 글자수 초과는 발견 즉시 고쳐야 하는 사안이다.

둘째, 완성도 문제다. 규정을 다 지켜도 상투적이고 AI가 쓴 것 같은 문장으로는 그 학생만의 특성이 드러나지 않는다.

이 단계는 두 문제를 순서대로 처리한다. 먼저 5대 기준으로 규정과 품질을 점검하고 수정한 뒤, 마지막 단계에서 AI 작문 패턴 교정 로직을 세특이라는 정형 서식에 맞게 골라 적용해 결과물이 인간 교사가 직접 쓴 것처럼 읽히도록 마무리한다.

**중요한 전제**: 세특은 명사형 종결(~함, ~보임), 주어 생략, 높은 명사 밀도가 규정으로 요구되는 정형 서식이다. 이 서식 자체를 흔드는 일반적인 자연스러움 교정(문장 종결형 다양화, 경어체 변주 등)은 적용하지 않는다. 대신 서식 안에서도 여전히 AI 흔적으로 드러나는 부분(상투어, 번역투, 대명사와 복수형 과다, 과장 어휘 군집, 명사 나열식 단조로움)만 골라 적용한다.

## 점검 5대 항목 요약

| 순서 | 항목 | 핵심 |
|---|---|---|
| 1 | 관찰자 시점 | 교사가 직접 본 행동, 발언, 산출물 근거 없이 내면 상태를 단정하지 않는가 (가장 중요, 신뢰도 문제) |
| 2 | 절대 기재 금지 | 논문, 도서출간, 지식재산권, 인증시험, 모의고사 성적, 교외대회, 해외활동, 가족 배경, 장학, 특정 기관/상호명, 자격증이 한 단어라도 있는가 (발견 즉시 삭제) |
| 3 | 서식 규정 | 명사형 종결, 주어 생략, 가운데점 등 특수기호 금지, 날짜 형식, 도서명 표기 규칙 |
| 4 | 분량 | 나이스 바이트 계산 기준으로 제한치 이내인가 |
| 5 | 개별화와 구체성 | 다른 학생에게 그대로 써도 어색하지 않은 공통문구는 아닌가, 서사 구조가 있는가 |

각 항목의 상세 기준, 감지 키워드, 예시는 `references/04-prohibited-and-format.md`에 정리되어 있다. 이 단계를 시작하기 전에 반드시 그 파일을 읽는다.

## 작업 흐름

### 1단계: 입력 형태 판별

2단계(작성)에서 만든 `세특초안_정리.xlsx`가 있으면 엑셀 배치 모드로, 학생 한 명의 텍스트만 다루는 경우면 텍스트 모드로 진행한다. 애매하면 사용자에게 물어본다.

### 2단계: 참조 문서 로드

`references/04-prohibited-and-format.md`를 로드해 기재금지 항목 표, 서식 규정, 글자수 기준, 개별화 기준을 확인한다.

### 3단계: 원본 분량 기록

수정에 들어가기 전에 원본 텍스트의 나이스 바이트 수를 먼저 계산해 기록해 둔다.

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/skills/seteuk-workflow/scripts/byte_counter.py "여기에 원본 세특 텍스트"
```

또는 파일로:

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/skills/seteuk-workflow/scripts/byte_counter.py --file 경로.txt
```

이 값은 9단계(분량 일치 확인)에서 최종본과 비교하는 기준이 된다.

### 4단계: 절대 기재 금지 사항 검사 (최우선)

`04-prohibited-and-format.md` 1항의 표를 기준으로 논문, 도서출간, 지식재산권, 인증시험 성적, 모의고사 성적, 교외대회 실적, 해외활동, 부모/가족 배경, 장학 관련, 특정 기관/상호명, 자격증 관련 서술이 있는지 문장 단위로 확인한다.

발견되면:
1. 해당 문구를 제거한다.
2. 그 문구가 뒷받침하려던 역량이나 특성을 교내 활동(발표, 실습, 프로젝트, 산출물)으로 대체할 근거가 문장 안에 이미 있는지 확인한다.
3. 대체 근거가 없다면 지어내지 말고 사용자에게 "이 부분을 어떤 교내 활동 근거로 대체할까요?"라고 되묻는다.

### 5단계: 관찰자 시점 검사

문장에 학생의 내면 상태나 역량을 나타내는 서술어("느낌", "깨달음", "관심이 많음", "역량이 있음", "재능을 보임", "열정적임" 등)가 있는데, 같은 문장이나 앞선 문맥에 교사가 관찰 가능한 구체적 행동, 발언, 산출물 근거가 없다면 지적 대상이다.

- 문제없음: "발표 후 학생들의 질문에 즉석에서 답하는 모습을 보이며 자신감을 보임" (관찰 가능한 행동이 먼저, 그 결과로 특성 서술)
- 지적 대상: "평소 관심이 많고 열정적임" (무엇을 보고 그렇게 판단했는지 근거 없음)

구조는 [수업 중 구체적 행동, 발언, 태도] → [그로 인한 결과, 반응, 변화] → [교사 시선의 역량 평가] 순서여야 한다. 셀프 생기부 표현("나는", "~라고 느낌", "~라고 깨달음")도 같은 문제이므로 교사 관찰 시점 서술어(수행함, 기여함, 참여함, 조사함, 발표함)로 바꾼다.

교정 전략:
1. 관찰 가능한 행동을 먼저 서술하고 그 결과로 드러난 특성을 뒤에 배치한다(순서 재배치만으로 해결되는 경우가 많다).
2. 단정적 내면 서술을 실제로 있었던 관찰 가능한 행동으로 대체한다.
3. 근거 자체가 원문에 없다면 임의로 만들어내지 말고 사용자에게 "이 부분은 어떤 관찰 사실에 근거한 것인가요?"라고 되묻는다.

### 6단계: 서식 규정 검사

`04-prohibited-and-format.md` 2항 기준으로 확인한다.

- 명사형 종결(~함/~보임/~수행함)로 끝나지 않는 문장 → 명사형으로 변환 (예: "발표를 진행했다" → "발표를 진행함")
- 문장마다 반복되는 주어("학생은", "OOO는" 등) → 생략
- 가운데점, 물결표, 괄호형 특수기호 등 나이스에서 지양하는 기호 → 쉼표나 "와/과"로 대체
- 날짜 형식: 0000.00.00.-0000.00.00. (하이픈 연결, 8자리 숫자, 마침표 필수. 물결 사용 금지). 1~3학년 1학기 재임기간은 실제 개학일과 무관하게 매년 3월 1일부터로 통일
- 도서명 표기: 단순 독후감이면 도서명 입력 불가. 발표, 토론, 보고서, 콘텐츠 제작 등에 실제로 활용된 경우에만 (도서명, 저자명) 또는 도서명 단독 표기 가능. 텍스트에 도서 관련 서술이 있으면 이 조건을 충족하는지 확인하고, 충족하지 않으면 도서명을 빼고 "관련 도서를 참고하여" 정도로 일반화할지 사용자에게 확인

### 7단계: 분량(바이트) 검사

`04-prohibited-and-format.md` 3항의 계산 방식(한글 1자 = 3Byte, 그 외 1Byte)으로 계산한다. 직접 세지 말고 `scripts/byte_counter.py`로 정확히 계산한다.

계산된 바이트 수와 해당 영역의 제한치(참조 문서 3항 참조, 예: 과목별 세특 통상 1500Byte)를 비교해 초과 여부를 안내하고, 초과 시 압축을 제안한다. 제한치는 학년도와 영역마다 개정되므로 "정확한 최신 기준은 나이스 공지로 재확인 권장"을 결과에 반드시 덧붙인다.

### 8단계: 개별화와 구체성 검토

`04-prohibited-and-format.md` 4항 기준(구체성, 서사 구조)으로 개선 여지를 확인하고 제안한다.

- 공통문구 지양: 다른 학생에게 그대로 붙여도 어색하지 않은 뻔한 문장인지 확인
- 구체성: "열심히 함", "성실히 참여함" 같은 상투적 표현을 활동명, 산출물, 수치로 대체할 수 있는지 확인
- 서사 구조: 계기 → 활동 → 결과(산출물) → 확장(후속 탐구)의 흐름이 있는지 확인

원문에 없는 사실을 창작하지 않는다. 구체화할 정보가 부족하면 사용자에게 되묻는다.

### 9단계: 분량 일치 확인 (필수)

4~8단계 수정을 모두 반영한 초안이 나오면, 바로 다음 단계로 넘어가지 말고 3단계에서 기록한 원본 바이트 수와 비교한다. 목표는 나이스 제한치를 넘지 않으면서 원본과 비슷한 분량(원본 대비 ±10% 이내 권장)을 유지하는 것이다. 교사가 이미 그 길이로 칸을 채우기로 계획했을 가능성이 높으므로 규정 위반이 없는데도 임의로 크게 줄이거나 늘리지 않는다.

- 수정본이 원본보다 많이 짧아진 경우(기재금지 문구 삭제 등): 원문에 이미 있는 관찰 근거, 산출물, 과정을 더 살려 분량을 채운다. 채울 근거가 부족하면 사용자에게 되묻는다.
- 수정본이 원본보다 많이 길어진 경우(관찰자 시점 보강 등): 제한치 초과 위험을 먼저 확인하고, 핵심에서 벗어난 수식어나 중복 표현부터 압축한다.

조정 후 `scripts/byte_counter.py`로 바이트 수를 다시 계산해 최종 확정한다.

## 마무리 단계: 자연스러움 마무리

5대 항목 점검과 분량 조정이 모두 끝난 뒤, 마지막으로 결과물이 AI가 쓴 것처럼 읽히지 않는지 점검한다. 이 단계는 세특 서식 규정을 어기지 않는 범위 안에서만 진행한다.

**적용하는 패턴**:

- 상투적 어휘: "중요하다", "핵심적", "효과적", "혁신적" 같은 일반 서술어가 반복되면 구체적 표현으로 교체 (8단계 개별화 검토와 겹치는 부분은 중복 작업하지 않는다)
- 번역투: "~에 대해", "~을 통해", "~에 있어서" 같은 조사 번역투, "~을 가지고 있음" 같은 동사 잉여 표현
- 대명사, 복수형 과다: "이들", "그것들" 같은 불필요한 대명사와 복수 접미사 "-들" 남용
- 과장 어휘 군집: "다각도로", "심도 있게", "폭넓은" 같은 과장된 수식어가 몰려 있는 경우
- 명사 나열식 단조로움: "~에 대한 탐구, 분석, 정리를 수행함"처럼 명사만 나열되는 문장이 반복되면 문장 구조를 다양화 (단, 명사형 종결 자체는 유지)
- 쉼표 과다: 한 문장에 쉼표가 지나치게 많이 몰려 있는 경우

**적용하지 않는 패턴** (세특 서식 규정이 우선):

- 문장 종결형 다양화 — 세특은 명사형 종결이 규정이므로 대상 아님
- 주어 생략 완화 — 세특은 주어 생략이 규정이므로 대상 아님
- 경어체 변주 — 세특에 해당 없음
- 명사 밀도 낮추기 — 세특은 압축된 사실 기술이 목적이라 명사화가 표준

**의미 보존 원칙**: 이 마무리 단계에서도 4~8단계에서 확정한 사실 관계, 수치, 활동명, 산출물명은 절대 바꾸지 않는다. 목표는 표현을 자연스럽게 다듬는 것이지 내용을 재작성하는 것이 아니다. 원문의 70퍼센트 이상이 그대로 유지되는 선에서 다듬는다.

## 결과 제시 (텍스트 모드)

서브에이전트로 호출된 경우, 아래 형식 대신 `references/05-parallel-dispatch.md`에 지정된 학생별 출력 형식으로만 응답하고 오케스트레이터에게 결과를 돌려준다. 아래 형식은 오케스트레이터가 직접 이 문서를 따르는 경우(학생 수가 적어 서브에이전트를 쓰지 않는 경우)에 쓴다.

```
## 세특 검수 결과

**분량(바이트)**: 원본 N Byte → 최종본 M Byte / 제한 P Byte (한글 기준, 최신 기준은 나이스 공지로 재확인 권장)

### 1. 절대 기재 금지 사항
- [발견된 항목과 위치, 또는 "해당 없음"]

### 2. 관찰자 시점
- [근거 없는 단정 서술과 수정 내역, 또는 "해당 없음"]

### 3. 서식 규정
- [명사형 종결, 주어, 특수기호, 날짜 형식, 도서명 표기 위반과 수정 내역, 또는 "해당 없음"]

### 4. 개별화와 구체성
- [구체적 제안, 또는 "현재 서술로도 충분히 구체적임"]

### 5. 자연스러움 마무리
- [적용한 패턴과 수정 내역, 또는 "특이사항 없음"]

---

## 최종본

[모든 점검과 마무리를 반영한 세특 최종본]
```

텍스트가 처음부터 규정과 완성도를 모두 충족한다면 그대로 인정하고 "5대 항목 모두 이상 없음. 최종본은 원문과 동일함"이라고 안내한다. 불필요한 수정을 강요하지 않는다.

## 엑셀 배치 모드

2단계에서 만든 `세특초안_정리.xlsx`를 `scripts/batch_check.py`로 1차 스크리닝한다.

```bash
python3 ${CLAUDE_PLUGIN_ROOT}/skills/seteuk-workflow/scripts/batch_check.py <입력파일.xlsx> <출력파일.xlsx>
```

- 입력 파일은 최소 "이름"(또는 "학번")과 "세특" 두 컬럼이 있다고 가정한다. 컬럼명이 다르면 실행 전 Read로 파일 구조를 먼저 확인하고 `--name-col`, `--text-col` 옵션으로 지정한다.
- 출력 파일은 학생별로 원본, 바이트 수, 기재금지 발견사항, 관찰자 시점 의심 문장, 서식 위반(가운데점, 명사형 종결, 날짜 형식)을 나란히 정리한다.
- 스크립트는 키워드와 패턴으로 기계적으로 판별 가능한 항목만 자동 탐지한다. 개별화와 구체성 제안, 자연스러움 마무리, 애매한 판단(기관명인지 일반명사인지 등)은 사람이 직접 검토해야 하므로, 1차 스크리닝이 끝나면 `references/05-parallel-dispatch.md` 기준으로 학생을 그룹으로 나눠 서브에이전트에 병렬로 위임하거나(학생 수가 적으면 직접), 위의 3~9단계와 자연스러움 마무리 절차를 학생별로 적용한 뒤 최종본을 채워 넣는다.
- 서브에이전트로 위임할 때는 1차 스크리닝 출력 파일에서 해당 그룹 학생들의 행(원본, 바이트 수, 자동 탐지된 위반 사항)만 프롬프트에 담아 전달한다. 스크립트가 이미 찾아낸 위반 사항을 서브에이전트에게 알려주면 같은 항목을 처음부터 다시 찾지 않아도 된다.

## 산출물

`세특_최종.xlsx` 한 개(또는 요청에 따라 학생별 텍스트). 학생별로 원본 초안, 점검 결과, 최종본, 바이트 수를 함께 정리해 전달한다.

## 주의 사항

- 절대 기재 금지 항목과 관찰자 시점 문제는 규정과 신뢰도에 직결되므로 임의로 판단이 애매하면 반드시 사용자에게 확인을 구한다. 없는 사실을 지어내는 것보다 되묻는 편이 항상 낫다.
- 나이스 바이트 제한과 기재금지 세부 기준은 학년도마다 소폭 개정된다. 이 스킬의 기준은 통상적으로 유지되어 온 핵심 사항이며, 최종 확정은 해당 학년도 나이스 공지나 학교생활기록부 기재요령으로 재확인해야 한다는 점을 결과에 항상 안내한다.
- 글을 쓸 때 가운데점을 구두점으로 사용하지 않는다. 이는 점검 대상이기도 하지만, 이 워크플로우 자신의 출력에도 동일하게 적용된다.
