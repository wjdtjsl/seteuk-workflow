# Seteuk Workflow: 세특 작성 전 과정 자동화

폴더 하나에 학생 및 수업 자료를 모아두고 이 스킬을 실행하면, 필요한 정보만 질문으로 받은 뒤 아래 3단계를 순서대로 이어서 진행해 완성된 세특을 전달한다.

1. **자료 정리** — 형식이 제각각인 원자료를 표준 엑셀로 정리
2. **초안 작성** — 정리된 자료를 근거로 학생별 세특 문단 작성 (학생이 많으면 서브에이전트 병렬 처리)
3. **최종 점검** — 기재금지, 서식, 분량, 개별화, 자연스러움까지 검수하고 마무리 (학생이 많으면 서브에이전트 병렬 처리)

특정 교과나 학년에 종속되지 않는다. 국어, 수학, 영어, 과학, 사회, 예술, 체육, 기술가정, 정보, 제2외국어 등 어떤 교과든, 1학년부터 3학년까지 어떤 학년이든, 특수교육대상 학생이든 동일한 절차와 기준을 적용한다.

## 0단계: 시작 전 정보 수집

작업을 시작하기 전에 아래 정보가 갖춰졌는지 확인한다. 대화에 이미 답이 있다면 다시 묻지 않는다. 없다면 AskUserQuestion으로 한 번에 질문한다.

1. **대상 폴더**: 학생 자료와 교사 평가가 들어있는 폴더. 연결된 폴더가 없으면 폴더 연결을 요청한다.
2. **교과목명, 학년, 학기**: 세특 문체와 서식(재임기간 등)을 정하는 데 필요하다.
3. **나이스 바이트 제한**: 교사가 별도로 말하지 않으면 기본값 500자(약 1500바이트)로 진행한다고 안내한다.
4. **대상 학생 범위**: 폴더 안 전체 학생인지, 특정 학생만인지.
5. **세특 강조점(선택)**: 특정 역량이나 활동을 강조하고 싶은지. 없으면 생략 가능하다고 안내한다.

이 다섯 가지를 확인한 뒤에만 1단계로 넘어간다. 정보가 애매하면 작업을 진행하지 말고 먼저 물어본다 — 이는 이 워크플로우 전체를 관통하는 원칙이다. 세특은 지어내는 글이 아니라 기록하는 글이므로, 근거나 맥락이 부족한 채로 다음 단계로 넘어가지 않는다.

## 실행 흐름

세 단계와 병렬 처리 기준은 독립된 지식 블록으로 `references/`에 정리되어 있다(01: 자료 정리, 02: 초안 작성, 03: 최종 점검, 04: 기재금지/서식 상세 기준, 05: 서브에이전트 병렬 처리 기준). 각 단계에 진입할 때마다 해당 파일을 먼저 읽고 그 안의 절차를 그대로 따른다. 요약만 보고 넘어가지 않는다.

### 1단계: 자료 정리

`references/01-data-curation.md`를 읽고 절차대로 진행한다. 폴더를 스캔하고, 형식별로 텍스트를 추출하고, 학생 결과물과 교사 평가를 구분해 학생별로 귀속시키고, 신뢰도 낮은 항목은 "확인불가"로 표시한 뒤, 표준 스키마로 `학생자료 정리.xlsx`를 만든다. 이 문서에 명시된 사용자 확인 지점(폴더 스캔 결과 요약, 컬럼 설계 확인)은 생략하지 않는다.

### 2단계: 초안 작성

`references/02-writing.md`를 읽고 절차대로 진행한다. 1단계 산출물을 근거로, 0단계에서 수집한 교과목/학년/학기/바이트 목표를 반영해 학생별 세특 초안을 작성한다.

대상 학생 수를 센 뒤 `references/05-parallel-dispatch.md`의 기준으로 서브에이전트 사용 여부와 개수를 정한다. 학생 수가 적어 직접 처리하는 경우가 아니라면, 학생을 그룹으로 나눠 Agent 도구로 서브에이전트를 병렬로 호출해 초안을 작성시키고 결과를 취합한다. 초안 작성 후 스크립트로 글자수와 금지 표현을 점검하고 `세특초안_정리.xlsx`로 저장한다.

### 3단계: 최종 점검과 마무리

`references/03-check.md`와 `references/04-prohibited-and-format.md`를 읽고 절차대로 진행한다. 2단계 산출물을 입력으로 받아, 관찰자 시점, 절대 기재금지, 서식 규정, 분량, 개별화와 구체성 순으로 점검하고 수정한 뒤, 마지막으로 자연스러움을 다듬는다.

이 단계도 2단계와 동일하게 `references/05-parallel-dispatch.md` 기준으로 서브에이전트 병렬 처리 여부를 판단한다. 2단계에서 이미 그룹을 나눴다면 같은 기준으로 다시 나누되, 그룹 구성이 반드시 2단계와 같을 필요는 없다. 결과는 `세특_최종.xlsx`로 저장한다.

각 단계가 끝나면 무엇을 했는지 한두 문장으로 짧게 요약하고(서브에이전트를 몇 개 썼는지 포함) 바로 다음 단계로 넘어간다. 다만 근거 부족, 확인불가 항목, 기재금지 대체 근거 필요 같은 판단이 애매한 지점이 생기면 그 즉시 멈추고 사용자에게 되묻는다 — 애매한 부분을 임의로 판단하고 계속 진행하지 않는다.

## 최종 산출물 전달

`세특_최종.xlsx`를 사용자가 볼 수 있는 위치에 저장하고 전달한다. 전달할 때 아래를 짧게 요약한다.

- 정리하고 작성한 학생 수
- "확인불가"나 "미확인"으로 남은 항목이 있으면 몇 건인지
- 기재금지 사항이 발견되어 수정한 건수
- 목표 바이트 대비 최종 분량 범위
- 작성과 점검 단계에서 서브에이전트를 몇 개 사용했는지

## 전체 원칙

- 모든 단계에서 학생이 실제로 하지 않은 일, 교사가 실제로 관찰하지 않은 내면 상태를 만들어내지 않는다.
- 근거가 부족하거나 판단이 애매하면 임의로 채우지 말고 사용자에게 되묻는다.
- 가운데점을 비롯한 특수문자는 세특 본문뿐 아니라 이 워크플로우 자신의 출력에서도 쓰지 않는다.
- 나이스 바이트 제한과 기재금지 세부 기준은 학년도마다 소폭 개정되므로, 최종 결과에는 항상 "정확한 최신 기준은 나이스 공지로 재확인 권장"을 덧붙인다.
